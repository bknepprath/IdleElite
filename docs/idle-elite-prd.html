Core Loop

The app is built for quick checks: choose a ready skill, spend stamina, bank progress, and leave with a clear next action.

30 sec to 5 min sessions
1. Pick Skill

Open a skill with stamina available or a goal worth pushing.

2. Run Activity

Tap one large activity button. The button owns cost, reward, time, rate, art, and progress.

3. Read Progress

Skill XP, stamina, mastery, and action completion are visible without extra explanation copy.

4. Rotate

When a pool is tired, swap skills or leave and return to offline progress.

Activity Button Rules

The activity button is the main game object. Keep it dense, tappable, and consistent with the Activity Database.

Source Data

activity-database.json owns names, unlocks, XP, stamina, seconds, rate, and art paths. The PRD should not duplicate the action list.

Card Contents

Show title, action art, background art, XP, stam, time, rate, mastery bar, medal state, and the running progress rail.

Copy Rules

Use terse labels: XP, Stam, Time, Rate. Avoid instructional text inside the phone UI.

Boost States

Motivation and ad boosts should be visible as small state badges, not separate menus.
